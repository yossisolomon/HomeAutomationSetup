# Home Automation Architecture Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stand up a self-documenting, diff-friendly automation authoring workflow (hybrid HA-YAML + Node-RED) plus a refreshed source-of-truth doc — before any real automation is written.

**Architecture:** HA YAML/blueprints for simple automations, Node-RED for complex flows (scaffolding deferred). A Python generator builds an autogenerated ToC in `docs/automations.md` from a required `# meta:` annotation on each automation; a versioned `core.hooksPath` pre-commit hook lints YAML, validates meta presence + unique names, and regenerates the ToC. A reusable Telegram call-to-action blueprint provides the "unsure → ask" primitive.

**Tech Stack:** Python 3.11 (pyenv env `homeautomation`) + pytest + PyYAML, `jq`, `yamllint`, bash git hooks, Home Assistant YAML/blueprints, Telegram bot.

**Working repo:** `/Users/yossi_solomon/dev/HomeAutomationSetup` (run all commands from here). Commits happen on the Mac; blacky pulls. HA config validation runs in the `homeassistant` docker container on blacky.

`scripts/gen_automations_toc.py`:
```python
#!/usr/bin/env python3
"""Generate / validate the automations ToC in docs/automations.md.

Usage:
  gen_automations_toc.py            # regenerate the AUTOGEN block in docs/automations.md
  gen_automations_toc.py --check    # validate meta presence + unique names; exit 1 on problems
"""
import re
import sys
import json
import pathlib

REPO = pathlib.Path(__file__).resolve().parent.parent
AUTOMATIONS = REPO / "config" / "automations.yaml"
SCRIPTS = REPO / "config" / "scripts.yaml"
FLOWS_DIR = REPO / "nodered" / "data" / "flows"
TOC = REPO / "docs" / "automations.md"

START = "<!-- AUTOGEN:automations START -->"
END = "<!-- AUTOGEN:automations END -->"

META_RE = re.compile(r"#\s*meta:\s*(.+?)\s*$")
INFO_META_RE = re.compile(r"meta:\s*(.+?)\s*$")
ALIAS_RE = re.compile(r'^\s*-?\s*alias:\s*(.+?)\s*$')
SCRIPT_KEY_RE = re.compile(r'^([A-Za-z0-9_][A-Za-z0-9_-]*):\s*$')


def parse_meta(s):
    out = {}
    for part in s.split(";"):
        part = part.strip()
        if not part or "=" not in part:
            continue
        k, v = part.split("=", 1)
        out[k.strip()] = v.strip().strip('"').strip("'")
    return out


class Entry:
    def __init__(self, name, engine, file, meta):
        self.name = name
        self.engine = engine
        self.file = file
        self.meta = meta


def parse_automations(path):
    entries = []
    if not path.exists():
        return entries
    last_meta = None
    for line in path.read_text(encoding="utf-8").splitlines():
        if line.lstrip().startswith("#"):
            mm = META_RE.search(line)
            if mm:
                last_meta = parse_meta(mm.group(1))
            continue
        am = ALIAS_RE.match(line)
        if am:
            name = am.group(1).strip().strip('"').strip("'")
            entries.append(Entry(name, "HA-automation", path.name, last_meta))
            last_meta = None
    return entries


def parse_scripts(path):
    entries = []
    if not path.exists():
        return entries
    last_meta = None
    current = None
    alias = None

    def flush():
        nonlocal current, alias, last_meta
        if current is not None:
            entries.append(Entry(alias or current, "HA-script", path.name, last_meta))
        current, alias, last_meta = None, None, None

    for line in path.read_text(encoding="utf-8").splitlines():
        if line.lstrip().startswith("#"):
            mm = META_RE.search(line)
            if mm and current is None:
                last_meta = parse_meta(mm.group(1))
            continue
        km = SCRIPT_KEY_RE.match(line)
        if km:
            saved_meta = last_meta if current is None else None
            flush()
            current = km.group(1)
            if saved_meta is not None:
                last_meta = saved_meta
            continue
        am = ALIAS_RE.match(line)
        if am and current is not None:
            alias = am.group(1).strip().strip('"').strip("'")
    flush()
    return entries


def parse_flows(dirpath):
    entries = []
    if not dirpath.exists():
        return entries
    for f in sorted(dirpath.glob("*.json")):
        try:
            data = json.loads(f.read_text(encoding="utf-8"))
        except Exception:
            continue
        nodes = data if isinstance(data, list) else data.get("flows", [])
        for n in nodes:
            if isinstance(n, dict) and n.get("type") == "tab":
                meta = None
                for ln in (n.get("info", "") or "").splitlines():
                    im = INFO_META_RE.search(ln)
                    if im:
                        meta = parse_meta(im.group(1))
                        break
                entries.append(Entry(n.get("label", "(unnamed)"), "NodeRED", f.name, meta))
    return entries


def collect():
    return parse_automations(AUTOMATIONS) + parse_scripts(SCRIPTS) + parse_flows(FLOWS_DIR)


def render_table(entries):
    header = ("| name | engine | file | intent | waf | mode |\n"
              "|------|--------|------|--------|-----|------|")
    if not entries:
        return header + "\n| _(none yet)_ | | | | | |"
    rows = []
    for e in sorted(entries, key=lambda x: (x.engine, x.name)):
        m = e.meta or {}
        rows.append("| {} | {} | {} | {} | {} | {} |".format(
            e.name, e.engine, e.file,
            m.get("intent", "—"), m.get("waf", "—"), m.get("mode", "—")))
    return header + "\n" + "\n".join(rows)


def replace_block(doc, table):
    text = doc.read_text(encoding="utf-8")
    pattern = re.compile(re.escape(START) + r".*?" + re.escape(END), re.DOTALL)
    new = START + "\n" + table + "\n" + END
    if not pattern.search(text):
        raise SystemExit("AUTOGEN markers not found in {}".format(doc))
    doc.write_text(pattern.sub(new, text), encoding="utf-8")


def check(entries):
    problems = []
    seen = {}
    for e in entries:
        if e.meta is None or not e.meta.get("mode"):
            problems.append("missing meta annotation: {} ({})".format(e.name, e.file))
        seen.setdefault(e.name, []).append(e.file)
    for name, files in seen.items():
        if len(files) > 1:
            problems.append("duplicate name '{}' in {}".format(name, ", ".join(files)))
    return problems


def main(argv):
    entries = collect()
    if "--check" in argv:
        problems = check(entries)
        if problems:
            print("ToC validation failed:", file=sys.stderr)
            for p in problems:
                print("  - " + p, file=sys.stderr)
            return 1
        return 0
    replace_block(TOC, render_table(entries))
    print("Regenerated {} ({} automations).".format(TOC, len(entries)))
    return 0


if __name__ == "__main__":
    raise SystemExit(main(sys.argv[1:]))
```

#!/usr/bin/env bash
# Normalize a Node-RED flow JSON for clean git diffs:
#  - drop volatile canvas coordinates (x, y) and editor-only fields
#  - sort object keys deterministically
# Usage: normalize_nodered.sh <file>   (prints normalized JSON to stdout)

file="${1:?usage: normalize_nodered.sh <flow.json>}"
jq -S 'walk(if type == "object" then del(.x, .y) else . end)' "$file"

- [ ] **Step 1: Implement the hook**

`scripts/git-hooks/pre-commit`:
```bash
#!/usr/bin/env bash
# Versioned pre-commit hook. Enable once with:
#   git config core.hooksPath scripts/git-hooks
set -euo pipefail
ROOT="$(git rev-parse --show-toplevel)"
cd "$ROOT"

# 1. Lint HA YAML (pyenv auto-activates via .python-version).
yaml_files=$(git diff --cached --name-only --diff-filter=ACM | grep -E '^config/.*\.ya?ml$' || true)
if [ -n "$yaml_files" ]; then
  echo "[pre-commit] yamllint…"
  python -m yamllint $yaml_files
fi

# 2. Normalize any staged Node-RED flow files in place, then re-stage.
flow_files=$(git diff --cached --name-only --diff-filter=ACM | grep -E '^nodered/data/flows/.*\.json$' || true)
if [ -n "$flow_files" ]; then
  echo "[pre-commit] normalizing Node-RED flows…"
  for f in $flow_files; do
    tmp="$(mktemp)"
    bash scripts/normalize_nodered.sh "$f" > "$tmp"
    mv "$tmp" "$f"
    git add "$f"
  done
fi

# 3. Validate meta annotations + unique names.
echo "[pre-commit] validating automation meta + names…"
python scripts/gen_automations_toc.py --check

# 4. Regenerate the ToC and stage it.
echo "[pre-commit] regenerating docs/automations.md…"
python scripts/gen_automations_toc.py
git add docs/automations.md

echo "[pre-commit] OK"
```

**Files:**

- [ ] **Step 1: Write the guide** with these exact sections and content:

`docs/automation-architecture.md`:
~~~markdown
# How We Build Automations

## Engine choice
- **HA YAML / blueprint** — single trigger -> single action or notification, no shared
  runtime state. Lives in `config/automations.yaml` (or a blueprint instance).
- **Node-RED** — two or more inputs fused, timers/debounce, a state machine, or
  cross-domain coordination. Lives in `nodered/data/flows/<domain>.json`.

When in doubt, prefer HA YAML; promote to Node-RED only when the YAML gets awkward.

## Naming
Every automation has a **globally unique alias**, domain-prefixed
(`climate-…`, `boiler-…`, `presence-…`). kebab-slug or human-readable both fine — pick
one and stay consistent. The alias is the ToC key; the commit hook fails on duplicates.

## Required `meta` annotation
Every automation MUST carry a meta annotation (the commit hook enforces this):
- **HA YAML** — a comment directly above the entry:
  `# meta: intent="…"; waf=low|med|high; mode=auto|cta`
- **Node-RED** — a line in the flow tab's `info` field:
  `meta: intent="…"; waf=…; mode=…`

`mode=auto` = fully automated. `mode=cta` = sends a call-to-action and waits for the
human. `waf` = Wife-Acceptance-Factor risk if it misfires.

## Call-to-action over fragile automation
When the system can't be *sure* it should act (window state unknown, safety risk like
lowering blinds), **ask instead of act**. Use the Telegram CTA blueprint
`config/blueprints/script/homeassistant/telegram_confirmable.yaml`: it sends a message
with Confirm/Dismiss inline buttons and runs the chosen action on `telegram_callback`.
This is the default for any `mode=cta` automation.

## Self-documenting ToC
`docs/automations.md` holds an autogenerated table built by
`scripts/gen_automations_toc.py` from the meta annotations. Never hand-edit the table;
edit the annotation at the source. The pre-commit hook regenerates and stages it.

## Diff hygiene (Node-RED — activated in the climate flagship spec)
Node-RED stores everything in JSON with volatile canvas coordinates and opaque node ids.
To keep diffs reviewable:
- **`node-red-contrib-flow-manager`** splits flows into per-domain files under
  `nodered/data/flows/` (verify module name/maintenance before install).
- **`scripts/normalize_nodered.sh`** strips `x`/`y` coords and sorts keys; the pre-commit
  hook runs it on any staged `flows/*.json`.
- **Externalize Function-node JS** into `nodered/data/lib/*.js`
  (`functionExternalModules: true` in `settings.js`) and lint with ESLint. *(ESLint
  install is deferred until the first `lib/*.js` exists.)*
- **Persistent global context** via `contextStorage` (file store) in `settings.js` for
  shared runtime vars (`presence`, `outdoor_aqi`, `co2`).

## Validating changes
- `make lint` — yamllint over `config/`.
- `make toc` — regenerate the ToC.
- `make check` — full HA config check (runs in the `homeassistant` container on blacky).
~~~
