# How We Build Automations

## Engine choice
- **HA YAML / blueprint** — single trigger -> single action or notification, no shared
  runtime state. Lives in `config/automations.yaml` (or a blueprint instance).
- **Node-RED** — two or more inputs fused, timers/debounce, a state machine, or
  cross-domain coordination. Lives in `nodered/data/flows/<domain>.json`.

When in doubt, prefer HA YAML; promote to Node-RED only when the YAML gets awkward.

## Naming
Every automation has a **globally unique alias**, domain-prefixed
(`climate-…`, `boiler-…`, `presence-…`). kebab-slug or human-readable both fine — pick
one and stay consistent. The alias is the ToC key; the commit hook fails on duplicates.

## Required `meta` annotation
Every automation MUST carry a meta annotation (the commit hook enforces this):
- **HA YAML** — a comment directly above the entry:
  `# meta: intent="…"; waf=low|med|high; mode=auto|cta`
- **Node-RED** — a line in the flow tab's `info` field:
  `meta: intent="…"; waf=…; mode=…`

`mode=auto` = fully automated. `mode=cta` = sends a call-to-action and waits for the
human. `waf` = Wife-Acceptance-Factor risk if it misfires.

## Call-to-action over fragile automation
When the system can't be *sure* it should act (window state unknown, safety risk like
lowering blinds), **ask instead of act**. Use the Telegram CTA blueprint
`config/blueprints/script/homeassistant/telegram_confirmable.yaml`: it sends a message
with Confirm/Dismiss inline buttons and runs the chosen action on `telegram_callback`.
This is the default for any `mode=cta` automation.

## Self-documenting ToC
`docs/automations.md` holds an autogenerated table built by
`scripts/gen_automations_toc.py` from the meta annotations. Never hand-edit the table;
edit the annotation at the source. The pre-commit hook regenerates and stages it.

## Diff hygiene (Node-RED — activated in the climate flagship spec)
Node-RED stores everything in JSON with volatile canvas coordinates and opaque node ids.
To keep diffs reviewable:
- **`node-red-contrib-flow-manager`** splits flows into per-domain files under
  `nodered/data/flows/` (verify module name/maintenance before install).
- **`scripts/normalize_nodered.sh`** strips `x`/`y` coords and sorts nodes by id; the
  pre-commit hook runs it on any staged `flows/*.json`.
- **Externalize Function-node JS** into `nodered/data/lib/*.js`
  (`functionExternalModules: true` in `settings.js`) and lint with ESLint. *(ESLint
  install is deferred until the first `lib/*.js` exists.)*
- **Persistent global context** via `contextStorage` (file store) in `settings.js` for
  shared runtime vars (`presence`, `outdoor_aqi`, `co2`).

## Validating changes
- `make lint` — yamllint over `config/`.
- `make toc` — regenerate the ToC.
- `make check` — full HA config check (runs in the `homeassistant` container on blacky).
