## General conventions

### Commits
- Small, focused commits — one logical change per commit.
- Conventional Commits format: `type(scope): summary`.
- Validate `docker-compose.yml` before committing: `docker compose config`

### Docker Compose
- All services in `docker-compose.yml` at repo root.
- Pin image tags to `stable` or a specific version — never `latest` in production.
- Each service's persistent data under its own top-level folder.
- Named bind-mounts only (`./service/data:/container/path`), no anonymous volumes.

### Security
- Never commit secrets. Use `config/secrets.yaml` (gitignored).
- Never expose the stack to the internet without HTTPS + auth.

### Adding a new service
1. Add service block to `docker-compose.yml`.
2. Create `<service>/` directory for data + seed config.
3. Update **Services** table in `README.md`.
4. Add conventions to relevant `docs/` file.
